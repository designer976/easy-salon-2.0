Componentes mestre da Easy 3.0

Este guia documenta os tokens e componentes presentes na home (home.html). Toda nova tela ou ajuste deve reutilizar estas variáveis e classes — sem inventar cores, fontes ou tamanhos novos. Um script Python (validate_components.py) valida automaticamente se as páginas seguem esta regra.

Efeitos & animações

Três efeitos assinatura da home: revelação no scroll (BlurFade), troca de palavra com blur por letra, e o glow rotativo nas pills especiais.

BlurFade no scroll

Elementos entram com opacity: 0 → 1, filter: blur(10px) → 0 e translateY(18px) → 0 em 1.4s. Disparado por IntersectionObserver com stagger de 90ms entre irmãos da mesma seção. 

/* CSS */
.fade {
  opacity: 0; filter: blur(10px); transform: translateY(18px);
  transition: opacity 1.4s, filter 1.4s, transform 1.4s
              cubic-bezier(0.22, 1, 0.36, 1);
}
.fade.is-in { opacity: 1; filter: none; transform: none; }

/* JS */
new IntersectionObserver((entries) => {
  entries.forEach(e => e.isIntersecting && e.target.classList.add('is-in'));
}, { threshold: 0.12 }).observe(el);

Cycling word (blur por letra)

Cada letra entra com blur + translateY, com stagger de ~40ms. Sai espelhada antes de a próxima palavra entrar. Usado no hero (“Sua recepção 24h para…”).

Sua recepção 24h para 
<span class="cycle-word">
  <span class="cycle-word__inner" data-cycle='["salões","barbearias","estúdios"]'></span>
</span>

/* CSS */
.cycle-word__char {
  opacity: 0; filter: blur(10px); transform: translateY(6px);
  animation: cw-blur-in 0.55s cubic-bezier(0.16, 1, 0.3, 1) forwards;
  animation-delay: var(--cw-delay, 0s);
}

Regra & script de validação

Os componentes da home (home.html) são mestres. Toda nova página deve reutilizar tokens (--c-…, --ff-…, --shadow-…) e classes (.btn, .pill, .veja-card, etc) sem redefinir.

Como rodar: a partir da raiz do projeto, execute
python3 validate_components.py
O script analisa todos os .html e reporta:
  • cores hex/rgb fora da paleta (não-tokens)
  • uso de famílias de fonte fora do sistema (Inter, Arial, Roboto…)
  • classes-mestre redefinidas (override de .btn-primary, .pill-accent, etc)
  • componentes que existem só na home mas foram “copiados” em outras páginas
